# Структура директорий

Документ описывает назначение директорий проекта и правила размещения новых файлов.

## Корень проекта

### docs

Документация по архитектуре, структуре и контрактам данных.

### src

Исходный код приложения.

### dist

Результат production сборки. Создаётся командой `npm run build`. Файлы внутри не редактируются вручную.

### node_modules

Установленные зависимости проекта. Создаётся командой `npm install`. Файлы внутри не являются частью исходного кода.

## docs

### docs/architecture

Архитектурная документация проекта.

- `data-contracts.md` описывает канонические модели и форматы локального хранения
- `directories.md` описывает назначение директорий

## src

### src/app

Композиция приложения после загрузки профиля.

Здесь находятся маршруты, нижняя навигация и подключение функциональных модулей.

### src/app/styles

Стили компонентов уровня приложения. Сейчас содержит оформление нижней навигации.

### src/features

Функциональные модули приложения. Каждый модуль хранит собственные страницы, модели, данные, репозитории и локальные стили.

Новый код конкретной функции следует размещать внутри соответствующего модуля, а не в общей директории.

### src/features/messaging

Чаты и сообщения.

Отвечает за список чатов, экран диалога, отправку, редактирование, удаление, ответы, реакции, статусы сообщений и сохранение в Избранное.

### src/features/messaging/hooks

React состояние модуля сообщений.

Здесь находится единая точка изменения чатов и сообщений со стороны интерфейса.

### src/features/messaging/model

Канонические типы чата, сообщения и общего состояния мессенджера.

Модели не зависят от React и локального хранилища.

### src/features/messaging/pages

Маршрутные экраны списка чатов и переписки.

Страницы отображают данные и вызывают операции из хранилища сообщений.

### src/features/messaging/repository

Чтение, проверка, нормализация, миграция и сохранение данных мессенджера.

Только этот слой знает о ключах локального хранилища и старом формате чатов.

### src/features/onboarding

Регистрация нового пользователя и выбор интересов.

### src/features/onboarding/pages

Экран пошаговой регистрации.

### src/features/onboarding/styles

Стили регистрации.

### src/features/profile

Профиль пользователя, интересы и настройки.

### src/features/profile/data

Статические данные профиля. Сейчас содержит трёхуровневое дерево интересов.

### src/features/profile/model

Канонические типы профиля и настроек.

### src/features/profile/pages

Экран профиля, редактирование данных и панели настроек.

### src/features/profile/repository

Чтение, нормализация, миграция и сохранение профиля и настроек.

### src/features/waiting-room

Комната ожидания и локальное состояние поиска собеседника.

### src/features/waiting-room/assets

Изображения комнаты ожидания.

### src/features/waiting-room/pages

Экран комнаты ожидания.

### src/shared

Общий код без привязки к конкретному функциональному модулю.

В эту директорию следует помещать только то, что используется несколькими модулями или реализует базовую инфраструктуру.

### src/shared/lib

Небольшие универсальные функции.

Содержит работу с датами, идентификаторами, буфером обмена и безопасным приведением неизвестных данных.

### src/shared/icons

Локальная копия SVG иконок, используемых приложением.

Все экраны получают пути к иконкам через `index.ts`. Файл `README.md` содержит полный перечень, а `LICENSE` сохраняет лицензию Ionicons.

### src/shared/storage

Базовая работа с локальным хранилищем и версионированной оболочкой данных.

Файл этого уровня не знает структуру профиля или сообщения.

### src/styles

Глобальные стили и общие переменные интерфейса.

- `variables.scss` содержит переменные Ionic и базовые цвета
- `global.scss` содержит глобальные правила и служебные классы
- `messenger.scss` содержит общие стили экранов мессенджера

## Точки входа

### src/main.tsx

Запускает Ionic и React, подключает глобальные стили и монтирует приложение.

### src/App.tsx

Создаёт корневую оболочку Ionic и маршрутизатор.

## Конфигурационные файлы

### package.json

Зависимости и команды разработки, сборки и запуска на iOS.

### capacitor.config.json

Настройки Capacitor.

### ios

Нативный Xcode проект, созданный Capacitor.

Внутри находятся Swift точка входа, storyboard, ресурсы приложения, Podfile и скопированная web сборка. Обновление web ресурсов выполняется командой `npm run ios:sync`.

### vite.config.ts

Настройки сборщика Vite.

### tsconfig.json

Настройки проверки TypeScript для исходного кода.

### tsconfig.node.json

Настройки TypeScript для конфигурационных файлов Node.

### index.html

HTML точка входа Vite.

## Правила размещения

- Компонент и логика одной функции размещаются внутри одного модуля в `src/features`
- Канонические типы размещаются в `model`
- Доступ к сохранённым данным размещается в `repository`
- React состояние функции размещается в `hooks`
- Маршрутные экраны размещаются в `pages`
- Общие функции без бизнес логики размещаются в `src/shared`
- Глобальные стили размещаются в `src/styles`
- Сгенерированные файлы в `dist` и `node_modules` вручную не изменяются
